# Pipeline Redesign Plan: Modernizing for Efficiency and Robustness

## Current Pain Points
1. **Data Cleaning Redundancy (Brittleness):** The main data script (`pcs-data-processing.R`) missed the `family` variables. As a workaround, `pcs-analysis-modern.R`, `visualize_results.R`, and `kin_script.R` all separately re-read the raw `.dta` file to manually extract and clean `family3`, `family4`, and `family5`. If you need to change how `family` is coded, you have to find and change it in three different files.
2. **Computational Waste (Inefficiency):** Ordinal mixed-effects models (`clmm`) and Poisson mixed models (`glmer`) on thousands of observations take a long time to converge. Currently, these identical models are fitted repeatedly in the analysis script, the plotting script, and the kin comparison script. Tweaking a plot's axis label currently requires waiting minutes for the models to re-fit.
3. **Lack of Orchestration:** There is no explicit dependency graph. It is unclear what order scripts should be run in, increasing the chance of out-of-sync plots and tables.

## Proposed Redesign

We will enforce a strict separation of concerns by breaking the pipeline into four modular, numbered scripts, and orchestrating them with a `Makefile`. We will also create `data/` and `models/` directories to store intermediate objects.

### 1. `R/01_data_processing.R` (Data Munging)
- Read `pc1980-85-90-95.dta`.
- Perform **all** variable harmonization, including `family` and `friends` ordinal cleaning.
- Compute all Mundlak within-between variables and factor definitions.
- **Output:** Save the fully prepped dataset to `data/pcs_cleaned_mundlak.rds`.

### 2. `R/02_model_fitting.R` (Heavy Computation)
- Read `data/pcs_cleaned_mundlak.rds`.
- Fit the computationally expensive models: `m_friends`, `m_family`, and `m_orgs`.
- **Output:** Save the fitted model objects using `saveRDS()` into a new `models/` directory (e.g., `models/m_friends.rds`, `models/m_family.rds`, `models/m_orgs.rds`).

### 3. `R/03_make_tables.R` (Fast Reporting)
- Read `data/pcs_cleaned_mundlak.rds` for the descriptives table.
- Read the pre-fitted models from `models/`.
- Use `modelsummary()` to generate `Tabs/pcs_network_stability_modern.tex`, `Tabs/pcs_kin_comparison.tex`, etc.
- *Benefit:* Takes < 2 seconds. You can rapidly iterate on LaTeX formatting without refitting models.

### 4. `R/04_make_plots.R` (Fast Visualization)
- Read the pre-fitted models from `models/`.
- Extract coefficients, calculate standard errors, and render `ggplot2` visualizations (e.g., `Plots/unified_cultural_effects.png`).
- *Benefit:* Takes < 2 seconds. You can tweak colors, themes, and labels instantly.

### 5. `Makefile` (Orchestration)
- We will write a GNU `Makefile` to formally define these dependencies.
- Typing `make` in the terminal will execute the pipeline.
- If you edit `04_make_plots.R`, `make` knows it only needs to run the plotting script. If you edit `01_data_processing.R`, `make` will rerun everything downstream. This ensures your LaTeX derivables are always perfectly in sync with your code, without any wasted computation.
